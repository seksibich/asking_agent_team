# stock_agent_kit 文档中心

本目录是工程业务、数据、测试与运维文档的统一入口。面向普通使用者先读系统全景，面向开发者按改动范围进入对应专题。

## 推荐阅读顺序

1. [系统全景与审查结论](01-系统全景与审查结论.md)：系统做什么、数据怎么流转、本轮审查结论与风险等级。
2. [Agent 编排与业务模块](02-Agent编排与业务模块.md)：主 Agent、五个分析角色、量化选股、盯盘、情绪与回测闭环。
3. [前端业务与全时段规则](03-前端业务与全时段规则.md)：盘前、竞价、交易、午休、收盘、非交易日的页面和请求规则。
4. [数据存储、缓存与一致性](04-数据存储缓存与一致性.md)：数据库事实源、缓存边界、原子性、fallback 与准确性保障。
5. [测试、探针、监控与运维](05-测试探针监控与运维.md)：自动化测试、`/live`、`/ready`、Docker 和阿里云监控。

## 协议与权威索引

- [Agent 数据服务使用协议](AGENT_SERVICE_GUIDE.md)：鉴权、端点、响应、版本协调和错误处理。
- [服务业务索引](SERVICE_INDEX.md)：功能分组、Skill 与现行定时任务交叉表。
- [部署说明](../DEPLOY.md)：本地 Docker、裸机和阿里云部署。
- [数据库说明](../service/db/README.md)：表结构与数据库维护入口。

## 文档维护规则

新增或修改业务时，代码提交前必须同步对应专题文档、接口参数、数据口径、交易时段、fallback、测试矩阵和监控项。具体清单见 `.kiro/steering/documentation.md`。

Agent 相关或 Agent↔服务交叉文档发生变化时，还必须同步提升 `agent/init.md` 的 `AGENT_DOC_VERSION`，并在 `profile/CHANGELOG-AGENT.md` 顶部新增版本记录。
