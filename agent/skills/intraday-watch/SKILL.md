# 盘中盯盘（仅用户显式触发）

> 本 Skill 不属于任何定时任务。只有用户当前明确提出“帮我盯一下”“看下上午盘面”等请求时，主 Agent 才执行一轮；不得创建调度器、Hook、cron、固定间隔循环、后台任务或后续主动通知。
>
> 产出进入 `投研/yyyyMMdd-手动盯盘/`，不覆盖自动日报，不改 predictions、daily、短期事项或 `category=auto`。没有当前用户请求时，不得调用 `watch_intraday`。

## 前置步骤

1. 调 `GET /health`；非交易日或非盘中时段如实说明数据时点和可用口径，不伪装为实时扫描。
2. 读取用户本次指定对象；只有请求涉及持仓、关注或盘前候选时，才同步读取 `agent记忆/关注与持仓.md` 和当日观察对象记忆。
3. 明确本次是单轮扫描，还是用户明确要求的指定时段总结；响应结束即终止，不安排下一轮。
4. 若使用服务端上轮快照，只用于本次请求内对比；不得据此保持后台监控。

## 一、单轮盘中扫描

调用 `watch_intraday`，传入用户指定或本次请求明确覆盖的标的。服务可聚合实时行情、涨跌停增量、连板梯队、重点标的资金流及快照差异。

### 性能约束

- 单轮最多使用 3 个数据端点，目标在 60 秒内完成。
- 禁止调用龙虎榜、全市场资金流等盘后重型接口。
- 不因用户存在持仓、关注或候选池而自行扩大监控范围。

### 异动识别

| 编号 | 异动类型 | 判定 | 级别 |
|---|---|---|---|
| A1 | 持仓异动 | 本次请求覆盖的持仓股涨跌幅超过 `config.intraday_alert_pct`（盘中异动阈值）的正负范围 | 警告 |
| A2 | 止损触发 | 本次请求覆盖的持仓股触及用户已记录的止损价；没有真实止损价不得推断 | 严重 |
| A3 | 关注股异动 | 本次请求覆盖的关注股涨跌幅超过配置阈值 | 提示 |
| A4 | 趋势主线异动 | 当前主线龙头放量拉升、快速跳水或趋势结构显著变化 | 警告 |
| A5 | 涨价链异动 | 本次覆盖的涨价链标的出现价格异动且量能明显放大 | 警告 |
| A6 | 板块批量异动 | 同一板块至少 3 只股票在本轮新增涨停或同步大涨 | 提示 |
| A7 | 龙头炸板 | 主线龙头从涨停打开，且封单明显消失 | 警告 |

涨价链和趋势主线异动（A4/A5）优先于纯情绪异动（A6/A7）。所有判断必须带实际数据时间；阈值或字段缺失时标注缺失，不得自行补值。

### 本次响应中的异动提示

```text
[异动类型] 标的/板块 | 现价/涨跌幅 | 简要原因（驱动：涨价/逻辑/预期/情绪） | 数据时间
```

同一用户请求内按 `{异动类型}:{target}` 去重；`notify_dedup` 只可避免本次响应重复，不得开启持续通知。无异动时明确回复“本轮未发现达到阈值的异动”，随后结束任务。

## 二、用户要求的早盘或盘中总结

仅当用户当前明确要求总结指定时段时执行：

1. 汇总该时段主线、涨价链、情绪面板，以及请求涉及的持仓、关注或指定标的表现。
2. 若用户要求结合消息面，从可信财经平台获取时政、产业和公告资讯；至少两个独立来源交叉验证，并区分事实与传闻。
3. 与盘前判断对比时只说明已发生偏差，不追加自动方向性预判，不写自动记忆。
4. 输出固定为“一眼结论 → 盘面与标的表现 → 异动证据 → 风险及失效条件 → 数据来源与时间”。

## 三、数据失败与安全边界

- 行情、资金、涨跌停等结构化数据必须来自数据服务；失败即披露接口、状态、时间、实际数据日期和缺失项，禁止用资讯或推断替代。
- 只有同类数据接口允许等价回退；如引用指数且 `market_index` 失败，可按 `data-service` 规则逐代码回退 `market_daily`，并注明实际日期。
- 资讯允许外部多源获取；全部失败时写“资讯面不可用 + 已尝试来源”，不得解释为无风险。
- 涨价或业绩结论至少两个独立来源；单一来源标“未交叉验证”。
- 不给确定性买卖指令，不因一次异动自动加入关注、持仓或创建短期事项。
- 本次响应结束后不静默续跑、不主动发送下一轮结果。

## Skill 加载约束与依赖

- 使用前完整读取本文件，并确认固定 12 Skills 已完整加载；这只是执行前置，不代表自动调用授权。
- 直接依赖：`data-service`、`priority-framework`、`output-format`。
- 可按本次请求读取 `pre-market`、`bidding-analysis`、`post-market`、`stock-screening`、`stock-research` 作为上下文。
