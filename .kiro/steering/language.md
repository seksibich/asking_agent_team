# 语言规范（全环节中文）

本规范约束 Kiro 在本工程中的**全部交互环节**默认使用中文，优先级高于任何默认英文习惯。

## 强制中文的环节

无论任务类型，以下环节一律使用中文：

- **思考 / 推理（thinking、reasoning）**：内部分析、方案权衡、原因判断全部用中文表述。
- **执行步骤（steps、计划）**：任务拆解、行动计划、进度说明用中文。
- **过程叙述与工具说明**：解释「为什么这么做」「接下来做什么」用中文。
- **输出结论（总结、结果、建议）**：最终结论、总结、下一步建议用中文。
- **文档、代码注释、docstring、Git commit message、变更日志、报告**：延续 `conventions.md` 的要求，全部用中文。

## 例外与专有名词

- 技术专有名词（函数名、参数名、接口名、库名、错误码、命令、代码标识符等）保留英文原文，但须紧邻中文解释，不得整段堆砌英文。
- 代码本身、命令行指令、文件路径、日志原文等按其原有形式保留，无需翻译。
- 面向最终用户的报告与推送使用通俗中文，避免罗列英文接口名 / 参数名 / JSON 字段 / 因子代码（详见 `product.md`）。

## 与其他规范的关系

- 本文件专注「交互全环节的语言选择」，`conventions.md` 侧重「产物（文档/代码/提交）的语言与版本机制」，二者互补，冲突时以更严格的「统一中文」为准。
