# MEMORY — 永久记忆

> 只保存永久有效或长期稳定的信息及专项索引。任务进度、业务状态、选股、持仓、问题和待办严禁写入。

## 专项记忆索引

- 用户稳定资料与长期偏好：读取 `USER.md`。
- 服务版本、文档同步、功能索引与接口约定：读取 `服务状态与能力.md`。
- 当前关注与持仓：涉及相关任务时读取 `关注与持仓.md`。
- 当前短期事项：只读取 `短期记忆/` 中与任务相关且未过期的文件；Coze 对应 `所有对话/主对话/recent_memory/`。
- 本地/通用记忆根默认为 `盯盘/agent记忆/`；Coze 中本文件位于右侧记忆 `所有对话/主对话/MEMORY.md`。

## 永久规范

- 选股风格：涨价 > 逻辑 > 预期炒作 > 情绪；PE/PB 仅作风险背景。权威细则以当前 `priority-framework` 为准。
- 数据纪律：禁止编造；涨价和业绩结论至少两个独立来源交叉验证；关键数据标注来源与时间。

## 写入门禁

写入前必须同时确认：跨任务长期有效、不是当前业务状态、不是进度/问题/待办、已有多个独立样本验证、写明适用范围与失效条件。任一条件不满足则写入 `短期记忆/`、业务报告或审计记录；用户资料写入 `USER.md`，不得混入本文件。
