# 产品说明（stock_agent_kit）

## 一句话定位

一套与平台无关的「短线盯盘 + 投研」智能体初始化工具包，外加一份自带前后端 + 数据库的本地数据服务。智能体读取 `init.md` 完成自我初始化，通过 HTTP 调用数据服务获取行情/资金/新闻/财务数据，按固定分析重心产出盯盘与投研结论。

## 两大组成部分

1. **Agent 工作规范（提示词工程）**
   - 负责智能体的初始化、后续版本升级、任务编排、Agent 团队编排。
   - 核心文件：`init.md`（初始化入口）、`index.md`（强制执行索引）、`SOUL.md`（人格铁律）、`schedule.md`（定时任务）、`agents/`（主 + 5 子 Agent 团队）、`skills/*/SKILL.md`（12 个技能）、`memory/`（记忆规范）。
   - 版本机制：`AGENT_DOC_VERSION`（文档版本）+ `CHANGELOG-AGENT.md`（变更日志），智能体据此逐版本同步升级。

2. **数据服务（前后端 + DB 一体）**
   - 后端 `service/`（FastAPI），功能脚本 `skills/*/scripts/*.py`（`@register` 自动注册），前端 `web/`（同源 `/ui/`），数据库 DDL `service/db/`。
   - 统一调用协议：`POST /call {function, params}`，鉴权头 `X-API-Key`。
   - 数据源：tushare pro（15000 积分档）+ 东财等。
   - 版本自感知：每个响应带 `data_version`（功能索引哈希），新增功能自动改版本。

## 核心分析重心（贯穿全部工作，不可覆盖）

> **涨价 > 逻辑 > 预期炒作 > 情绪**

- 涨价（第一优先）：真实产品/商品涨价、供需反转，允许自主到行业平台/期货取数并 ≥2 来源交叉验证。
- 逻辑：产业链传导、景气拐点，以预期驱动为主，不以过往业绩为主（业绩披露期除外）。
- 预期炒作：政策/事件/题材催化，评估兑现概率。
- 情绪：连板/涨停/活跃度，仅作节奏与仓位。
- PE/PB 权重为 0，仅作风险提示中的过往估值背景。

## 三条数据红线（最高铁律）

1. **禁止编造**：行情/财务/资金/新闻/价格必须来自数据服务或可核验来源，拿不到就说拿不到。
2. **必须交叉验证**：涨价、业绩类结论 ≥2 独立来源；单一来源标「未交叉验证」。
3. **标注来源与时间**：每个数据点注明来源功能与获取时间。

## 关键运行机制

- **数据服务禁止降级**：数据类接口失败则失败，如实披露，不得用推断/编造填补。
- **资讯类允许多源**：新闻/资讯类可从各财经平台（新闻联播、财经媒体、外部搜索）交叉获取。
- **数据优先、结论优先**：报告固定为「一眼结论（核心摘要）→ 目录导读 → 详细正文」，先给仓位/次日倾向、题材/事件 Top N、最大风险/证伪。
- **选股回测闭环**：自动候选经 `log_selection(category=auto)` 登记，`selection_backtest` 跟踪 1/3/7/30 日收益并给调参建议；因子权重可配置、可留痕、可回滚。
- **Agent 团队**：主 Agent + 5 子 Agent；重量级任务（盘前/复盘/周月回测/用户分析）启用团队，盯盘由主 Agent 单跑。

## 面向用户的表达要求

面向用户的报告首屏、结论、正文、推送与表头统一使用通俗中文，直接说明「发生了什么、影响谁、为何关注、何时失效」；不得堆砌英文接口名、参数名、JSON 字段或因子代码。技术名称仅在数据来源附录、故障诊断或用户明确要求的参数说明中出现，并紧邻中文解释。
