# 盘中盯盘（主 Agent 模式）

> 盯盘追求时效，**由主 Agent 单跑，不启用团队**（团队仅用于盘前/盘后复盘/用户分析）。

> **输出目录取决于触发来源**（见 output-format）：定时任务(T3/T4/T5)盯盘异动/早盘总结进日期目录（`03-早盘总结.md`）并按需推送；**用户手动触发（如「帮我盯一下」「看下上午盘面」）→ 产出进 `投研/yyyyMMdd-手动盯盘/`，不写自动记忆与快照**。

## 前置步骤

1. **交易日守卫**：`GET /health` 的 `trade_open=false` 直接返回。
2. **强制读取当日观察对象记忆** `agent记忆/daily/yyyyMMdd.md`：用户持仓、关注股、相关板块、当日自动选股、当日重点板块。**持仓与关注及其相关板块必须重点盯**。
3. **读取上轮快照**：服务端维护 `intraday_snapshot`（首轮不存在则跳过对比）。

---

## 一、盘中扫描（09:35~11:30 / 13:05~14:55）

调 `watch_intraday`，传入监控标的清单。服务端一次返回：实时行情、涨跌停增量、连板梯队、重点标的资金流，并与上轮快照对比。

### 性能约束
- 盘中单轮 ≤ 3 个数据端点（`watch_intraday` 内部已聚合），单轮 ≤ 60 秒
- 禁止盘中调用重型接口（龙虎榜、全市场资金流），这些留到盘后

### 异动识别（满足任一即推送）

| 编号 | 异动类型 | 判定 | level |
|---|---|---|---|
| A1 | 持仓异动 | 持仓股涨跌幅超 ±{config.intraday_alert_pct}% | warning |
| A2 | 止损触发 | 持仓股触及止损价 | critical |
| A3 | 关注股异动 | 关注股涨跌幅超阈值 | info |
| A4 | 趋势主线异动 | 当期主线龙头放量拉升/跳水 | warning |
| A5 | 涨价链异动 | 观察池涨价链标的异动放量 | warning |
| A6 | 板块批量异动 | 同板块 ≥3 只本轮新增涨停/大涨 | info |
| A7 | 龙头炸板 | 主线龙头由涨停开板、封单消失 | warning |

**重心提醒**：涨价链/趋势主线异动（A4/A5）优先级高于纯情绪连板（A6/A7）。推送时先报前者。

### 推送格式
```
[异动类型] 标的/板块 | 现价/涨跌幅 | 简要原因（驱动：涨价/逻辑/预期/情绪）
```
示例：
```
⚠️ [涨价链异动] 某某能源 600XXX.SH | 25.3 / +8.2% | 产品报价上调，量能放大（驱动：涨价）
🚨 [止损触发] 某某股份 000XXX.SZ | 11.2 / -10.4% | 跌破止损 11.25
```

### 去重
生成 key `{异动类型}:{target}`，服务端 `notify_dedup` 在 {config.dedup_window_minutes} 分钟内同 key 不重复推送。

### 预判记录
定时 T3/T4/T5 有明确正式方向性预判时可追加 `predictions.jsonl`（标注 driver）；纯事实通报不记录。用户手动盯盘默认 ephemeral，不写 predictions、daily 或观察池。

## 二、盘中消息 + 早盘总结（12:50，主 Agent）

1. **盘中消息**：从外部财经平台多源检索午间时政/产业/公告（资讯不在数据服务，见 data-service「资讯类外部获取」，≥2 来源交叉），评估对下午盘影响，修正盘前预判
2. **早盘总结**：上午盘面回顾（主线表现、涨价链进展、情绪面板-上午数据、持仓/关注股上午表现），与盘前预判对比标注偏差
3. 生成 `03-早盘总结.md`（标题用 `# 🌅 早盘总结 — yyyy-MM-dd`，开篇先给 `## 🎯 一眼结论`：📊 上午盘面定性、🔥 上午最强主线、⚠️ 下午风险/关注点，再展开回顾；主要章节按 output-format 的 emoji 约定加图标）；有消息影响则推送（level: info）

## 四、静默原则

**无异动绝不打扰**：不生成文件、不发通知、不主动输出，仅更新服务端快照。

## Skill 加载约束 / 依赖 Skills

- 盘中任务启动前完整读取本文件并确认固定 12 Skills（含 `stock-research`）已完整加载；不得只凭 schedule 摘要执行。
- **直接依赖**：`data-service`、`priority-framework`、`output-format`。
- **协同 Skills**：`pre-market`、`bidding-analysis`（观察清单与预判输入）、`post-market`（盘中事实交接）、`stock-screening`（主线标的）、`stock-research`（用户单股调研引用盘中事实时协同，不作为定时必执行绑定）。
- T3/T4/T5 必须点名 `skills/intraday-watch/SKILL.md` 与 `skills/data-service/SKILL.md`；需要报告时同时点名 `skills/output-format/SKILL.md`。

## 盘中新闻与行情 fallback

T4 资讯（新闻/时政/公告）不在数据服务，从各财经平台多源检索（≥2 来源交叉，标来源与时间），全部失败标“资讯面不可用 + 已尝试来源”，不得推断“无风险”。**数据类行情或聚合接口失败则失败**、标 `degraded`/缺失来源、禁止编造，非关键接口失败不阻塞早盘总结；401/配置错误不盲目重试。若正文引用 `market_index`，按 `skills/data-service/SKILL.md` 的逐 code `market_daily` fallback 与实际日期标注执行。
