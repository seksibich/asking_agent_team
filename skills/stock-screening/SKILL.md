# 趋势 + 行业逻辑选股

## 选股哲学

自上而下，以**涨价链 → 行业趋势 → 个股**为路径，重逻辑轻情绪。**不做无基本面支撑的纯连板打板。**

## 流程

### 1. 定主线（涨价/逻辑优先 + 板块动量）
- 调 `price_hike_scan` 找当期涨价方向（第一优先）
- 调 `screen_sector` 做**板块轮动量化排名**（板块动量因子，见 quant-screening），锁定趋势向上的强势行业
- 调 `sector_dc` 看当日资金/涨跌确认
- 结合 `观察池` 已跟踪的趋势线索

### 2. 行业内选股
调 `screen_trend`（底层复用 `factors.py` 趋势侧因子），在选定行业/主线内初筛：
- 趋势硬过滤：多头排列（price>ma20>ma60）
- 因子排序：`mom_12_1` + `trend_ma` + `high_52w` 为主，低波动/低换手辅助
- 景气/涨价关联：主营与涨价链/景气主线相关（Agent 侧叠加）
- 排除：ST/退、高位纯情绪票

### 3. 四维、趋势与主线复核
对候选按 `priority-framework` 打分，涨价+逻辑维度至少一项 ≥0.7 才进强关注；同时核对量化综合分及关键因子、所属板块/产业链、板块短中期动量/量能/阶段，以及其与当前主线的关系（核心/分支/补涨/非主线）。业绩增长不能单独替代四维和趋势判断。

### 4. 正式候选理由链
每只正式趋势候选必须按固定顺序说明：**量化信号 → 板块趋势 → 当前主线关系 → 涨价/逻辑/预期催化 → 情绪与择时 → 风险/证伪**。不得只给趋势分、量化分或四维分；任一环没有可核验资料，必须在对应位置写「无可核验证据」，不得省略或臆造。

### 5. 产出选股报告
写 `选股/yyyyMMdd-趋势选股.md`，执行 `skills/output-format/SKILL.md` 的四维打分表、趋势主线表、量化分组表和「正式候选综合理由表」：
```markdown
# 趋势选股 — YYYY-MM-DD
## 核心摘要
## 目录导读
## 数据状态与口径
## 选股主线（涨价/行业逻辑说明）
## 趋势主线跟踪
## 正式候选综合理由表
## 分组解读
## 风险与证伪
## 数据来源
```
正式候选表逐只保留：量化综合分与关键因子、四维分、板块/产业链、板块短中期动量/量能/阶段、主线关系、催化与炒作路径、完整理由链、情绪与择时、风险与证伪。

### 6. 记忆
只有独立通过本正式量化/趋势流程的标的，才按触发来源和既有规则写 `predictions.jsonl`、观察池或 `log_selection`。业绩增长参考池不是正式选股，其成员不得因进入参考池而持久化；同一股票若独立通过正式流程，只能由本流程正常持久化。

## 严谨要求
- 技术面筛选数据来自 `screen_trend`，禁止编造
- 涨价/景气归因需可核验：涨价数据可自主到行业披露平台/期货行情取数并 ≥2 来源交叉验证，标注来源与时间
- **以预期驱动选股**，不以过往业绩为主（业绩披露期除外）
- **PE 不作为看多依据**，仅在「风险」列作过往估值背景（如是否透支预期）

- 数据或新闻缺失时按 `skills/data-service/SKILL.md` 降级并标 `degraded`，不得用技术分替代缺失的涨价/逻辑证据。
