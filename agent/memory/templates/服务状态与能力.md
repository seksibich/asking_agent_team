## 通用接口约定

- `GET /health`：健康与版本基线，字段位于顶层；每次任务开场刷新本文件，持仓版本除外。
- 其他业务 JSON 响应：成功或失败都先读取嵌套 `health`，比较文档、部署、功能、标签和持仓五轨版本，再处理业务数据或错误。顶层 `data_version` 仅作旧客户端兼容。
- 旧服务响应缺少 `health` 时，本次请求链最多补调一次 `/health`；补调失败只标“版本状态暂不可核验”，不得覆盖原业务结果。
- 同一任务以目标五轨版本元组作为一次性协调锁；升级或刷新请求返回相同元组时不得再次触发。
- `GET /functions`：当 `data_version` 变化时全量覆盖功能索引。
- `POST /call`：统一功能调用；业务数据位于响应 `data`，但必须先消费 `health`。
- 401/403 仍可消费响应 health；权限阻塞的刷新写入有时效 `短期记忆/`，完成后删除，不得提前标记同步完成。
- 鉴权只引用环境变量名，不得在本文件保存真实 Key。
- 接口失败产生的待办写入 `短期记忆/`，解决后删除。
