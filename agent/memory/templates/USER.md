## 写入与更新规则

- 只写用户明确表达或反复确认的稳定称呼、表达偏好、风险偏好、研究偏好和长期约定；记录依据与最近确认时间。
- 用户临时指令、单次语气或未经证实的行为推断不得提升为稳定偏好；需要跨轮次处理时写入短期事项。
- 当前持仓与关注写 `关注与持仓.md`；选股、行情、报告、工作进度、问题和待办写对应业务文件或短期记忆。
- 严禁保存真实 Key、Token、密码、身份证件、联系方式等敏感信息。
- 用户明确修改或撤回偏好时立即更新或删除旧项，不保留与当前意愿冲突的画像结论。
