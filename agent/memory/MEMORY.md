# 记忆治理规范

> 本文件是仓库内的记忆治理规则；运行期主记忆是逻辑文件 `MEMORY.md`，不得与本文件混淆。

## 运行期记忆结构

本地/通用环境默认位于 `盯盘/agent记忆/`：

```text
agent记忆/
├── MEMORY.md                 # 仅永久规范、稳定偏好、已验证经验及专项索引
├── USER.md                   # 用户明确表达或反复确认的稳定资料与长期偏好
├── 关注与持仓.md             # BASE_URL、portfolio_version、portfolio_get 全量镜像
├── 服务状态与能力.md         # 其余 health 版本、文档同步、功能索引与接口约定
├── 短期记忆/                 # 一事一文件及关联附件；解决、证伪或过期即删除
├── daily/                    # 按日业务快照，不属于主记忆
├── predictions.jsonl         # 预判审计镜像，权威记录在 DB
└── 学习日志-yyyy年MM月.md    # 回测与配置变更审计，不直接视为永久经验
```

Coze 必须严格分栏：左侧“工作文件”存放日报、投研、选股、周报、月报、复盘和 `tmp/`；右侧“记忆”中，`基础设定/SOUL.md` 保存人格，`所有对话/主对话/` 保存 `MEMORY.md`、`USER.md`、`关注与持仓.md`、`服务状态与能力.md`，其 `recent_memory/` 等价于逻辑 `短期记忆/`。平台 `SECRET.md` 或密钥区仅由平台安全机制管理，任何普通记忆不得保存真实密钥。

## 主 MEMORY 与 USER 准入规则

`MEMORY.md` 只能写跨任务长期稳定的信息：持久规范、稳定约定、经过多个独立样本验证且已抽象的通用经验，以及专项文件读取索引。每条注明首次记录时间、最近验证时间、适用范围、依据和失效条件。

`USER.md` 只保存用户明确表达或反复确认的称呼、风险偏好、表达偏好和长期约定；临时指令、当前持仓、选股、任务进度、问题、待办、敏感信息及未经证实的推断不得写入。

严禁把任务进度、当前选股/行情/持仓、单次结论、故障现场、待办、临时路径、服务版本或接口清单写入主 MEMORY。学习日志结论只有通过上述门禁后才可提炼；业务事实和处理过程不得复制进去。

## 短期记忆与关联附件

- 每个事项单独写入 `短期记忆/YYYYMMDD-HHmm-有效至YYYYMMDD-HHmm-描述.md`；Coze 写入 `recent_memory/`，文件名不变。
- 必填 `task_id`、创建/更新时间、失效时间、来源、状态、待处理事项、完成标准、删除条件和 `related_files`；时间统一为 Asia/Shanghai。
- 为解决该事项保存的接口报错、响应摘录等辅助材料可放同一短期目录，文件名必须复用事项完整前缀，如 `...-描述-接口报错.txt`，并以相同 `task_id` 和 `related_files` 双向关联。
- 开工时只读当前任务相关且未过期的条目；完成、证伪或过期后，立即删除事项文件及全部关联附件，不得长期堆积“已完成”记录。
- 旧 `观察池.md` 不再使用；有效线索迁入带时效的短期事项，无效内容删除。

## 工作文件与普通临时产物

普通临时脚本、临时文档、格式转换和中间产物只能写入工作文件根的 `tmp/tmp_YYYYMMDD-HHmmss_文件名`；本地默认即 `盯盘/tmp/`，Coze 位于左侧工作文件的 `tmp/`。它们不得进入任何记忆目录。仅为处理某个报错而保留且需要跨轮次读取的证据，才按上一节进入短期记忆并关联事项。

## 专项文件读写

- 每次开场读 `MEMORY.md`、`USER.md` 与 `服务状态与能力.md`；调用 `/health` 更新除持仓外的版本，文档、功能或标签版本变化时刷新对应章节。
- 涉及关注/持仓时读 `关注与持仓.md`；比较 `/health.portfolio_version`，不一致或本地版本为空时调用 `portfolio_get` 全量覆盖。
- `daily/`、predictions、学习日志、报告和选股 DB 都是业务快照/审计，只按任务读取，绝不复制到主 MEMORY 或 USER。
- 服务或上传失败产生的待处理事项写短期记忆；恢复后删除事项与附件，不得污染持仓镜像。
- 主 Agent 是共享记忆唯一写者；子 Agent 只回传结构化结果。
