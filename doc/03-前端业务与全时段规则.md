# 前端业务与全时段规则

## 时间基准

前端业务日期统一使用服务端健康快照中的上海日期和市场阶段；健康信息暂缺时才使用 `Asia/Shanghai` 计算，不使用浏览器所在时区决定交易日。后端仍会校验日期和模式，前端规则不能代替服务端门禁。

## 八类市场状态

| 状态 | 含义 | 盘中行业模式 | 情绪默认日期 |
|---|---|---|---|
| 盘前 `preopen` | 开盘前 | 自动切最近完整日 | 当天，标临时 |
| 集合竞价 `call_auction` | 竞价阶段 | 自动切最近完整日 | 当天，标临时 |
| 上午交易 `morning` | 连续竞价 | 可请求当天盘中叠加 | 当天，标临时 |
| 午间休市 `lunch` | 午休 | 自动切最近完整日 | 当天，标临时 |
| 下午交易 `afternoon` | 连续竞价 | 可请求当天盘中叠加 | 当天，标临时 |
| 收盘待确认 `closed_pending` | 收盘但日终未完成 | 自动切最近完整日 | 当天，标临时 |
| 日终完成 `final` | 当天完整数据可用 | 最近完整日 | 当天，标完整 |
| 非交易日 `non_trading_day` | 周末或休市 | 最近完整交易日 | 最近交易日 |

## 业务请求规则

- 量化选股：读取最近完整日预计算数据，不把盘中价格写入完整日因子。
- 行业分析：只有上午/下午连续竞价允许 `intraday`；其他阶段切换为 `latest_complete` 并提示用户。
- 情绪：交易日默认当天，非交易日默认最近交易日；根据阶段显示“临时”或“完整收盘”。
- 选股看板：默认目标交易日及之前三个交易日；查询与行情刷新分离。
- 量化盯盘：仅管理员可见。省略日期时展示最近有数据日，盘后、次日和非交易日均可回看最近 30 个自然日内的聚合消息；显式选择历史日期后断开 WebSocket，点击“最新”恢复实时模式。进入实时模式建立长连接，离开页断开浏览器连接，不停止服务端扫描。
  - **临时禁用（当前生效）**：前端设 `QUANT_WATCH_DISABLED = true`（`service/web/app.js`），强制隐藏量化盯盘入口与面板、不建立 WebSocket，管理员默认落地页回退到情绪温度。此开关只影响前端展示，**服务端确定性扫描、`quant_watch_status` 接口与 WebSocket 端点均不变**；恢复时把该开关改回 `false` 并 bump 静态资源版本即可。
- 顶部业务顺序固定为：量化盯盘、情绪温度、行业分析、量化选股、选股看板、回测分析、预计算、自选管理、配置管理。盯盘临时禁用期间，管理员登录后默认进入情绪温度；访客看不到管理员盯盘、预计算与自选管理，默认进入情绪温度；自选管理为管理员专属的顶部独立页签（不再并入配置管理二级入口）。
- 顶部页签横向可滚动（不再换行居中），页签采用按钮样式并使用与普通按钮区隔的独立配色；按钮按功能分色：主操作（蓝）用于查询/保存/登录，次操作（描边灰）用于刷新/取消/导出，强调操作（青）用于运行选股与增量预计算等耗时计算，危险操作（红）用于删除/移除。所有按钮按下有轻度按压、松开回弹并带点击波纹。
- 深色主题：整体仿 PC 交易软件夜间样式，统一圆角与卡片模块化。涨跌统一为稍深的红（上涨）/绿（下跌），平盘为白偏灰；同一股票的“最新价”与“最新涨幅”配色一致。大卡片不再显示 eyebrow 小标签，说明类文本整体降权以节省版面，并适配手机设备。
- `/ui/` 的 HTML、JavaScript 和 CSS 每次导航均向服务端重验证，发布时同时更新静态资源版本参数，避免浏览器缓存旧默认页逻辑。
- 回测：读取当前权限可见类别；访客不能看到关注和持仓。
- 自选、预计算、盯盘设置：管理员专属，服务端再次校验权限。

## 局部刷新体验

行情刷新调用 `/selections/quotes`，只更新当前列表的价格、行情标签和缺失选股价，不重新调用看板查询，不刷新整页，并保留筛选、排序和展开状态。刷新期间显示局部进度，成功后更新刷新时间并弹出 Toast；请求序号防止旧响应覆盖新响应。

行业、情绪、回测、预计算和量化盯盘均使用各自按钮状态表达加载过程。默认界面使用“最近更新、数据待确认、正在积累”等业务语言，不展示任务 ID、原始运行 ID、内部来源码或计算版本。

## 凭据与隐私

- 服务地址与访问凭据均缓存在 `localStorage`，便于手机浏览器（如 Safari）免重复登录；同时写入 `sessionStorage` 兼容旧逻辑。`forgetConnection()` 提供清除入口。这是为移动端便利做出的取舍：凭据在本设备长期可读，仅适用于自用本地面板；每次导航仍会向服务端重新校验角色权限，服务端鉴权为最终安全边界。
- 动态访客 Key 的列表只展示掩码；完整值仅创建成功时显示一次。
- 自选备注不作为 UI 内容；编辑已有记录时后端旧备注会被保留，避免无意覆盖。
- WebSocket 使用一次性短期票据，票据在数据库原子消费，日志会脱敏查询参数。

## 自动化测试矩阵

| 范围 | 自动检查 |
|---|---|
| 市场时钟 | 交易日七个阶段 + 非交易日最近交易日 |
| 前端状态 | 八类阶段、行业盘中限制、情绪默认日期、盯盘最近有数据日/历史日期/实时连接隔离、历史日期上限 |
| 页面契约 | JavaScript 引用元素存在、HTML 编号不重复、前端功能已注册、主标签顺序与管理员/访客默认页 |
| 局部刷新 | 不使用整页重载、不重查 `selection_dashboard`、存在局部进度 |
| 凭据缓存 | 访问凭据缓存到 `localStorage`（移动端免重复登录）并保留 `sessionStorage`、提供清除入口；访客 Key 列表不提供明文复制 |
| 语法 | `node --check service/web/app.js` 与 `market-state.js` |

测试只证明参数决策和静态契约；真实行情覆盖、浏览器视觉效果和云端网络仍需 Docker/预发布烟测。
