# 单只股票主动调研

## 适用与输出

- 用户主动提出「研究/分析/评估/看看 XX 股票」时启用，主 Agent 组织基本面、技术面为主，宏观与情绪协同。
- 固定输出：`投研/yyyyMMdd-{股票名}个股调研/研究报告.md`；同日同名重复时目录后加 `-2`、`-3`。
- 这是用户主动研究，不等同于自动选股，不因结论强弱改变记忆边界。

## 默认 ephemeral 边界（强制）

- 默认只生成报告：不写 `predictions.jsonl`、`daily/yyyyMMdd.md` 或短期事项，不调用 `log_selection`，不回测。
- 仅当用户明确说「加入观察」「持续跟踪」等持久化指令时：先调用 `portfolio_stock_search` 并从结果确认标准股票，在内存中形成 watch 草稿，调用 `portfolio_upload`；上传成功后用响应 rows 刷新 `关注与持仓.md`。只有用户另行明确要求纳入后续历史回测时，才调用 `log_selection(category=watch)`。
- watch 历史快照只进入 1/3/7/30 日观察性回测，必须与 auto 分组展示；不得进入自动胜率、`tuning_hints`、因子权重或情绪参数调优。
- `category=auto` 仅允许调度器正式自动候选使用；主动调研标的即使高分也不得登记为 auto。

## 团队分工

- **主 Agent**：确认标的、统合证据、二次核验、裁决结论与记忆边界。
- **基本面分析师（主责）**：主营/产品/订单/公告、行业与产业链位置、利好利空和兑现概率。
- **技术面分析师（主责）**：个股短中期趋势、量价、资金、量化分位与关键因子。
- **宏观分析师（协同）**：政策、产业事件、供给扰动、外部来源及时间窗口。
- **情绪分析师（协同）**：题材热度、阶段、市场情绪、建议出手权重与择时风险；内部参数名不得直接进入用户报告。
- **回测分析师**：默认不参与；用户明确持久化后仅处理 watch 观察性回测，禁止用于 auto 调参。

## 研究流程

1. **标的与数据口径**：核对代码、名称、交易日、实际数据日期、接口状态、fallback 与缺失字段。
2. **动态题材/具体事件**：结合外部财经资讯/时政/公司公告（多源检索，见 data-service「资讯类外部获取」，≥2 来源交叉）、`hot_dc`、`hot_ths`、`hot_kpl_list`、涨停连板、板块涨幅、量能资金，按“实体+动作+产品/政策+时间窗口”聚合同义事件。
3. **受益关系核验**：用主营、产品收入、订单、公告、客户/供应链与产业链位置证明直接或间接受益；纯名称命中、同名联想或蹭概念不得进入强候选结论。
4. **行业与产业链**：给出上中下游位置、供需、竞争格局、价格/订单传导及该股弹性。
5. **利好利空**：分别列可核验事实、来源时间、兑现路径、兑现概率和反向证据；传闻标「传闻，待证实」。
6. **四维评分**：按 `priority-framework` 输出涨价/逻辑/预期/情绪及硬门槛，不用 PE/PB 作为看多依据。
7. **量化与趋势**：内部读取原始量化分、当批横截面排名（0~100）和关键因子，判断短中期趋势、量价、资金与相对强弱；用户报告只展示中文指标名称、排名位置和结论，原始字段名移入数据附录。
8. **统一综合选股分**：`利好程度×0.35 + 题材热度×0.25 + 量化横截面分位×0.40`，三项均为 0~100；内部原始标准化分必须先转换为当批横截面排名。四维硬门槛继续生效。
9. **情绪择时**：内部消费情绪温度、题材阶段和出手权重参数；用户报告写成“建议出手权重”和“择时后评分”，只作概率性参考，不暴露内部参数名。
10. **风险证伪**：写清事件失效、订单/价格/政策落空、趋势破位、资金退潮等可观察条件。

## 报告固定结构

```markdown
# {股票名}个股调研 — YYYY-MM-DD
## 一眼结论（核心摘要）
- 仓位/次日倾向与建议出手权重：
- 重点题材/具体事件 Top N：
- 题材/事件 → 该股：直接/间接受益及核心证据：
- 最大风险/证伪：

## 目录导读
## 数据状态与口径
## 动态题材/具体事件及受益关系
## 行业、产业链与竞争位置
## 可核验利好、利空与兑现概率
## 四维评分
## 量化综合分、横截面分位与关键因子
## 个股短中期趋势、量价与资金
## 情绪择时与择时后评分/出手权重
## 风险证伪与未验证项
## 数据来源与时间
## 持久化状态
```

首屏必须使用 output-format 的「题材/事件—个股首屏结论表」；正文至少包含四维打分表、数据来源表。内部原始量化分只用于计算当批横截面排名，用户正文展示中文指标名称、排名位置和解释，不直接展示内部字段名或因子代码。

## 数据降级与严谨要求

- 关键数据不可用时保留章节并标 `degraded`、失败接口、fallback、重试轨迹、实际数据日期与缺失字段；不可把缺失解释为低热度或无风险。
- 所有事件、公告、订单、价格、资金与趋势数据标来源和时间；关键涨价/业绩结论按统一规则交叉验证。
- 利好/利空均为概率判断，不给确定性收益承诺；PE/PB 仅作风险背景。
- 报告末尾必须使用中文说明记录状态：默认写“记录状态：仅本次报告（未加入观察）”；用户明确要求持久化后写“记录状态：已加入观察（仅作观察性回测，不参与系统自动调参）”，并列出实际执行动作。不得把 `ephemeral`、`watch`、`auto` 等内部类别直接展示给用户。

## Skill 加载约束 / 依赖 Skills

- 使用前完整读取本文件并确认固定 12 Skills 已完整加载；本 Skill 不替代其他研究、取数或输出规范。
- **直接依赖**：`data-service`、`priority-framework`、`output-format`、`industry-analysis`、`quant-screening`、`stock-screening`。
- **协同 Skills**：`pre-market`、`bidding-analysis`、`intraday-watch`、`post-market` 提供时段事实；`review-learning` 仅在用户明确持久化为 watch 后执行隔离的观察性回测。
## v2.2.0 当前调度与日终边界

- 现行 Agent 定时任务仅为 T1/T3/W1/M1/P1；本 Skill 仍仅由用户主动单股调研触发，不加入任何定时任务，也不承接旧 T6/T7/D1。
- 服务端在交易日 16:00 自动完成日终收口；Agent 只读 `health.daily_finalize` / `precompute_status`，不得自动调用 `precompute_daily_factors`。
- 管理员预计算仅可在用户当前明确要求诊断或补数时单次手动调用，不得因单股数据缺失、调研失败、Hook、cron 或 Agent 循环触发。
