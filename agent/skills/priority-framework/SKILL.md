# 四维打分框架：涨价 > 逻辑 > 预期炒作 > 情绪

本框架是 Agent 一切分析的排序基准。对任何标的或板块，按四个维度分别评分，再加权汇总。

## 维度定义与评分标准

### 维度 1：涨价（权重 0.40，最高）

真实的产品/商品价格上涨或供需反转，是最硬的信号。

| 分值 | 判定标准 |
|---|---|
| 0.9~1.0 | 主力产品显著涨价（>10%），供需持续紧张，权威价格数据/期货行情持续印证 |
| 0.6~0.8 | 产品开始涨价或提价预期明确，有权威价格数据支撑 |
| 0.3~0.5 | 涨价预期存在但尚未落地，或仅局部品种 |
| 0.0~0.2 | 无涨价逻辑或价格下行 |

**数据来源**：不限于 tushare。允许并鼓励 Agent 自主到**行业披露平台、网络渠道、期货行情**获取价格数据（生意社、百川盈孚、上海有色网 SMM、卓创资讯、中国化工网、各行业协会，以及大商所/郑商所/上期所期货主力行情等），公司提价公告为佐证。**需 ≥2 独立来源交叉验证**，单一来源标注「未交叉验证」。

### 维度 2：逻辑（权重 0.30）—— 以预期驱动为主

产业链传导、景气拐点、涨价的**预期与前瞻**，需可验证。**优先看预期与产业链高频信号（价格、开工率、订单/排产），而非已落地的财务业绩**。

| 分值 | 判定标准 |
|---|---|
| 0.9~1.0 | 产业趋势明确向上，涨价/景气预期强且有高频数据印证，产业链上下游共振 |
| 0.6~0.8 | 逻辑清晰、景气向上，预期兑现路径可见（价格/订单前瞻支持） |
| 0.3~0.5 | 逻辑成立但预期兑现有不确定性 |
| 0.0~0.2 | 逻辑牵强或纯概念 |

> **预期驱动原则**：平时以「预期 + 高频产业数据（尤其价格）」评估逻辑，**不把已披露的过往业绩作为主要依据**；仅在**业绩预告/财报披露期**，业绩兑现度的权重临时提升，用于验证或证伪前期预期。

### 维度 3：预期炒作（权重 0.20）

政策、事件、题材催化，需评估兑现概率。

| 分值 | 判定标准 |
|---|---|
| 0.9~1.0 | 政策级强催化 + 首次发酵 + 高兑现概率 |
| 0.6~0.8 | 明确事件催化，题材新鲜 |
| 0.3~0.5 | 反复炒作的旧题材，或兑现概率低 |
| 0.0~0.2 | 纯蹭概念、无实质催化 |

### 维度 4：情绪（权重 0.10，辅助）

连板高度、涨停家数、活跃度，仅用于节奏与仓位。

| 分值 | 判定标准 |
|---|---|
| 0.9~1.0 | 市场高潮，赚钱效应强，标的处于主升 |
| 0.6~0.8 | 情绪回暖，梯队健康 |
| 0.3~0.5 | 情绪分歧或退潮初期 |
| 0.0~0.2 | 冰点或标的处于高位退潮 |

## 综合评分

```
总分 = 涨价×0.40 + 逻辑×0.30 + 预期炒作×0.20 + 情绪×0.10
```

- 总分 ≥ 0.75：强推荐关注（需涨价或逻辑维度至少一项 ≥ 0.7）
- 总分 0.55~0.75：值得跟踪；创建带失效时间、复查动作和删除条件的短期记忆条目
- 总分 < 0.55：暂不纳入

**硬约束**：若涨价与逻辑两项均 < 0.4（即仅靠预期+情绪），无论总分多高，标注为「情绪主导，高风险」，不进入强推荐。

## 打分表输出模板（报告中使用）

```markdown
| 标的 | 涨价 | 逻辑 | 预期 | 情绪 | 综合 | 结论 |
|---|---|---|---|---|---|---|
| XXX 某某股份 | 0.9 | 0.8 | 0.5 | 0.6 | 0.79 | 强推荐（涨价主导） |
| XXX 某某科技 | 0.2 | 0.4 | 0.9 | 0.8 | 0.49 | 情绪主导，高风险 |
```

每个分值后必须能对应到具体数据/事实依据，禁止拍脑袋打分。评分依据在报告正文逐条说明。

## 估值指标（PE/PB）的定位

- **不将 PE 作为选股打分因子，权重视为 0**。低 PE 不等于上涨动能，高 PE 也不否定预期驱动的行情。
- PE/PB 只反映**过往业绩对应的估值水平**，是历史指引，**不能推断股价上涨动能**。
- PE/PB 仅出现在报告的**风险提示**中：例如「当前 PE 处历史高位/已透支预期」「估值与预期背离」等风险语境，作为背景参考。
- 成长/预期驱动标的允许阶段性高估值，只要涨价/景气预期未被证伪，不因 PE 高而扣分。

## 候选综合评分与择时叠加（自动/用户主动双入口统一）★

四维评分用于价值与风险裁决，**四维硬门槛始终生效**；候选横向排序则对调度器自动选股和用户主动方向选股统一使用百分制：

```
综合选股分 = 利好程度×0.35 + 题材热度×0.25 + 量化横截面分位×0.40
择时后评分 = 综合选股分 × buy_weight_hint
```

- 三项输入均为 0~100：
  - **利好程度**：结合事件强度、直接/间接受益证据、兑现概率及利空反证；
  - **题材热度**：综合消息密度、热榜、涨停连板、量能和资金，题材不限传统行业板块；
  - **量化横截面分位**：在当批候选内计算 percentile。原始 score 若为 z-score，必须先转换，禁止直接与百分制混加。
- **自动入口**：调度器先动态识别题材/具体事件并映射候选，再执行 `screen_quant` / `screen_trend`，按“题材/事件 → 个股”分组后统一评分；符合正式自动候选定义者才可登记 `category=auto`。
- **用户主动入口**：用户可指定具体事件、行业导向或热门板块，走同一映射、筛选、评分和分组流程；形成正式候选后登记 `category=manual`，保存选股时价格、热点事件、主线地位和量化因子快照。
- `manual` 可做 1/3/7/30 日隔离回测，但必须与 auto 分开，不进入自动胜率、`tuning_hints` 或任何因子/情绪调参；普通研究和未通过正式门槛的线索仍为 `ephemeral`。
- **四维硬门槛不因综合选股分或择时放宽**：涨价与逻辑均 <0.4 时仍标“情绪主导，高风险”；强关注仍要求涨价或逻辑至少一项 ≥0.7。

### 择时来源
调 `market_timing`（基于情绪温度序列，识别连续冰点/高热）与 `sentiment_temperature`，得到：
`stance`（仓位倾向）、`buy_weight_hint`（出手买入权重系数）、`cold_streak`/`hot_streak`。

### 择时对出手的调节
- **连续冰点（cold_streak≥2）**：超跌/抄底窗口，提高 buy_weight_hint，优先有涨价/逻辑支撑且四维过门槛的标的，分批试仓。
- **连续高热（hot_streak≥2）**：警惕退潮，下调 buy_weight_hint，不追高，必要时给出降仓或空仓倾向。
- **中性**：按综合选股分和四维门槛排序，仓位适中。
- 择时后评分与建议出手权重均为概率性参考，不构成确定性买卖指令。

### 重仓 / 空仓环境判断（盘前、复盘用）
结合 **情绪（连续冰点/高热）+ 消息面 + 行业催化（含机构放话/研报观点）** 做初步判断：
- **具备重仓环境**：情绪健康或冰点反转 + 明确正向催化（涨价/政策/机构一致看多）+ 主线清晰
- **需要空仓/降仓**：连续高热见顶迹象 / 重大利空 / 无主线且情绪退潮 / 催化落空
判断写入盘前汇总与复盘报告，作为当日仓位上限的依据（非确定性指令）。

## Skill 加载约束 / 依赖 Skills

- 使用前必须完整读取本文件并确认固定 12 Skills（含 `stock-research`）已加载，不得以 `index.md` 或角色摘要中的“四维打分”简化描述代替。
- **直接依赖**：`data-service`（评分事实与 fallback）、`output-format`（打分与来源表）。
- **协同 Skills**：`industry-analysis`、`stock-screening`、`quant-screening`、`stock-research` 提供研究/候选；`pre-market`、`bidding-analysis`、`intraday-watch`、`post-market` 应用排序；`review-learning` 验证 auto 与隔离的 watch 效果。
- 任一评分依据因服务降级而缺失时，按 `skills/data-service/SKILL.md` 标 `degraded` 和缺失来源，不得用估算值补齐。
## v2.2.0 当前调度与日终边界

- 现行 Agent 定时任务仅为 T1/T3/W1/M1/P1；所有任务裁决使用现行编号，不使用旧 T6/T7/D1。
- 服务端在交易日 16:00 自动完成日终收口；Agent 只读 `health.daily_finalize` / `precompute_status`，状态异常时披露缺口，不得自动调用 `precompute_daily_factors`。
- 管理员预计算仅可在用户当前明确要求诊断或补数时单次手动调用，不得作为数据缺失、评分失败或门禁未通过时的回退。
